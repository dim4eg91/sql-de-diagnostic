# SQL/DE Diagnostic Prototype

Минимальный статический прототип диагностики готовности к SQL-собеседованиям и DE-практике.

Открытие локально:

```text
D:\stepik_projects\agents_01\sql_gym_prototype\index.html
```

Что есть сейчас:

- отдельная SQL-диагностика на 14 вопросов без Airflow/batch/Spark;
- отдельная DE-диагностика на 12 вопросов по DWH-слоям, Spark, Airflow, storage, DQ и контрактам;
- по 4 варианта ответа в каждом вопросе;
- перемешивание вопросов и вариантов при каждом новом запуске;
- двухшаговый ответ: сначала выбрать, потом подтвердить;
- подсветка зеленым/красным только после подтверждения;
- адаптация под мобильные экраны без горизонтального скролла и крупных скачков интерфейса;
- более правдоподобные неправильные ответы вместо очевидных заглушек;
- итоговый результат без дублирования в двух блоках;
- отдельный раздел с короткими карточками правил по SQL и DE;
- рекомендация SQL-маршрута на Stepik: базовый курс, Junior, Middle, программа Junior → Middle или программа Junior → Upper-Middle;
- рекомендация DE-маршрута на лендинг `https://kuzmin-dmitry.ru/de_practicum`;
- копирование результата в буфер обмена.

Публикация:

- приложение статическое, backend и сборка не нужны;
- самый быстрый вариант: Netlify Drop или Vercel Static Deploy, загрузить папку `sql_gym_prototype`;
- более правильный вариант для бренда: разместить на `kuzmin-dmitry.ru/sql-diagnostic` или `diagnostic.kuzmin-dmitry.ru`;
- для GitHub Pages нужно положить содержимое папки в репозиторий и включить Pages.

Важно: прототип ничего не сохраняет и никуда сам не отправляет данные. Реальная авторизация, тарифы, лид-хранилище и личный кабинет требуют backend.
